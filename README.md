# 甲骨文全文识别与AI推断系统

> 拓片整图识读 + 未知字AI破译 + 难以定义字大致意思揣摩 + 准确率优化

## 项目简介

本项目基于现有甲骨文识别技术（YOLOv8分割、ResNet/MobileViT分类），通过**四维优化策略**提升识别准确率和置信度，同时利用内置甲骨文知识库对污染字、残缺字、难以定义字进行可解释性AI推断和大致意思揣摩。

**核心创新**:
1. 从已有识别结果出发，通过上下文验证、多模型投票、字形校验、专家反馈四个维度持续提升准确率
2. **新增**：区分"污染字"和"难以定义字"，对非污染但难以精确定义的字进行大致意思揣摩
3. **新增**：内置141个部首/形符知识库、59组形近字、15类卜辞语境、65种常见合文
4. **新增**：支持上下文语境推断，根据前后已识别字辅助判断

## v3.0 更新内容

### 难以定义字揣摩功能
- 污染字检测：区分严重/中度/轻度污染
- 部首/形符分析：从字形特征推断所属类别
- 形近字参考：列出59组常见形近字供比对
- 语境推断：根据前后字判断卜辞类型（天气/祭祀/田猎/征伐等15类）
- 合文识别：检测65种常见合文（数字、庙号、月份、吉语等）
- 人性化解释报告：生成结构化分析文本

## 技术栈

| 组件 | 技术 | 用途 |
|------|------|------|
| **单字分割** | YOLOv8 | 拓片单字检测与定位 |
| **单字识别** | ResNet18/50 / MobileViT | 已知字分类识别 |
| **AI推断** | 内置知识库 + Qwen2.5-VL-7B | 未知字部首/象形分析 |
| **难以定义字揣摩** | 甲骨文知识库（规则引擎） | 非污染疑难字大致意思推断 |
| **后端服务** | FastAPI | RESTful API |
| **前端界面** | HTML5 + CSS3 + JS | 交互展示 |
| **准确率优化** | 多模型投票 + 上下文验证 | 提升结果可信度 |

## 快速开始

### 1. 安装依赖

```bash
cd backend
pip install -r requirements.txt
```

### 2. 启动后端服务

```bash

start_backend.bat

# 或PowerShell
.\start_backend.ps1

# 或直接运行
python main.py
```

服务启动后访问:
- API文档: http://localhost:8000/docs
- 健康检查: http://localhost:8000/health
- 知识库信息: http://localhost:8000/knowledge

用浏览器打开 `frontend/index.html` 即可使用。

## 系统架构

```
输入: 甲骨拓片图像/单字图像
        ↓
┌─────────────────────────────────────┐
│ 阶段1: 单字分割 (YOLOv8)             │
│ - 检测拓片中的每个单字位置           │
│ - 输出: 每个字的bounding box        │
└─────────────────────────────────────┘
        ↓
┌─────────────────────────────────────┐
│ 阶段2: 单字识别 (ResNet/MobileViT)   │
│ - 已知字 → 输出对应现代汉字         │
│ - 低置信度 → 进入疑难字分析         │
└─────────────────────────────────────┘
        ↓
┌─────────────────────────────────────┐
│ 阶段3: 疑难字分类                    │
│ ├─ 严重污染(置信度<20%) → 提示重拍  │
│ ├─ 中度污染(20-35%) → 提示+参考     │
│ └─ 难以定义(35-50%) → AI揣摩分析    │
└─────────────────────────────────────┘
        ↓
┌─────────────────────────────────────┐
│ 阶段4: AI揣摩未知字 (知识库+LLM)     │
│ ├─ 部首/形符分析（141个部首）       │
│ ├─ 形近字参考（59组形近字）         │
│ ├─ 语境推断（15类卜辞）             │
│ ├─ 合文检测（65种常见合文）         │
│ └─ 生成大致意思+建议                │
└─────────────────────────────────────┘
        ↓
┌─────────────────────────────────────┐
│ 阶段5: 准确率优化                    │
│ - 上下文语义验证                    │
│ - 多模型投票融合                    │
│ - 字形相似性校验                    │
│ - 专家反馈闭环                      │
└─────────────────────────────────────┘
        ↓
输出: 拓片全文 + 逐字释义 + 疑难字揣摩报告
```

## 难以定义字揣摩功能详解

对于常规拓片，系统不仅能检测污染字，还能对**字形清晰但不属于已知类**的难以定义字进行大致意思揣摩：

| 分析维度 | 方法 | 示例输出 |
|---------|------|---------|
| **污染评估** | 置信度分级 | "中度污染：部分笔画受污染或残缺" |
| **部首分析** | 141个部首/形符库 | "含「日」形，与太阳、日子相关" |
| **形近字** | 59组形近字对照 | "与「曰」「口」「白」字形相近" |
| **语境推断** | 15类卜辞常用字 | "结合上下文，属「天气」类卜辞" |
| **合文检测** | 65种常见合文 | "不排除是「大吉」合文" |
| **建议** | 综合建议 | "建议比对同一时期其他拓片" |

### API使用示例

```bash
# 单字识别（带上下文）

  -F "file=@character.jpg" \
  -F "context=癸卯卜贞今日"
```

```json
    "rough_meaning": "从字形特征看，含「日」形...",
    "explanation_text": "【字符分析报告】...",
    "possible_meanings": [...],
    "suggestions": [...]
  }
}
```

## API接口

### 1. 单字识别（支持上下文）

```
POST /predict
Content-Type: multipart/form-data

参数:
- file: 单字图片文件
- context: (可选) 前后相邻的已识别文字，用于语境推断

响应:
{
  "success": true,
  "character": "日" 或 "[?日?]",
  "confidence": 95.5,
  "is_unknown": false/true,
  "is_ambiguous": false/true,
  "is_polluted": false/true,
  "story": "字源故事...",
  "inference": { ... }  // 疑难字时返回
}
```

### 2. 拓片全文识别

```
POST /predict_full
Content-Type: multipart/form-data

参数:
- file: 整张拓片图片
- ordered_chars: (可选) 已确认的字符序列
```

### 3. 知识库查询

```
GET /knowledge
返回: 部首数量、分类、样本数据
```

### 4. 揣摩演示

```
GET /analyze_ambiguous_demo
返回: 一个难以定义字的完整揣摩分析示例
```

### 5. 字符列表

```
GET /classes
返回: 所有支持识别的字符列表
```

### 6. 健康检查

```
GET /health
返回: 服务状态、版本、功能列表
```
